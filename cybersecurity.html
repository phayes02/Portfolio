Ransomware

In our Cybersecurity class we were given two labs to recreate to present to the class. This lab is about ransomware, more specifically the WannaCry ransomware. In 2017, WannaCry begin to infect Windows systems in demand for payment of bitcoins. Like other ransomware, once the executable is ran your files will be encrypted and locked. The only way to get your files back is through payment. In this lab we install a repository called theZoo that contains the WannaCry files. Using Kali Linux, we will attack a Windows 7 virtual machine.

Slide 1: Showing theZoo repository after cloning the github repository

Slide 2: On the Windows 7 machine (Dummy) we go to the address of our Apache server. Then download the ransomware executable.

Slide 3: After running the executable our Dummy machine is now infected. The screenshot shows the screen of what you would see if you were infected by WannaCry.

Remote Access Trojan (RAT)

In Cybersecurity, the second lab that I was given is about Remote Access Trojans (RAT). Basically, RATs are a way for malicious users to remotely control your computer. A backdoor would need to be installed in the first place for this to work. For this lab, we will be using the Metasploit Framework, which contains a bunch of exploits for Windows, Linux, and Mac. Using Metasploit, we will run a persistence module. Note: In this example Metasploit 5 is being used, as of now Metasploit has been updated to its 6th version and the persistence module takes more to function. Again we will be using our Kali Linux machine to attack our Dummy machine.

Slide 1: After running the reverse tcp script, our system is listening and waiting for a victim to run the executable.

Slide 2: Its not shown, but the Dummy machine has now ran the executable. Now that we are connect to the Dummy machine we run the persistence modules and create the RAT. Now that the RAT is created, the machine is practically ours.

Slide 3: Since the RAT is in place, we can access the session at any given moment. After setting the configuration again, we now are back into the meterpreter waiting for the next commands.

TryHackMe

TryHackMe is a website to teach you about cybersecurity through their labs. In our Ethical Hacking class, we were tasked to complete all the learning paths labs that were free, complete at least one full path, and one of the network modules. It took a long time, but I was able to complete the task. If you are really interested learning about cybersecurity and other hacking tools, I'd strongly suggest looking into TryHackMe. It has a free plan, but you are limited on some things and all modules are not available.
